# /make-post — Single-Image Post Generator

**Date:** 2026-05-06
**Status:** Approved

## Overview

A new `/make-post` slash command that generates single-image social media posts tailored per platform. Takes a URL, blog post, GitHub repo, or topic and produces a striking image with text overlays plus an engaging caption. Supports visual mode (creative AI image) and text mode (branded text card).

Complements the existing `/make-carousel` (multi-slide) and `/make-reel` (video) commands.

## Command Interface

```
/make-post <url-or-topic> [--platform instagram|linkedin|x|all] [--mode visual|text] [--from-angle <path>] [--auto-publish platform]
```

**Defaults:** `--platform all`, `--mode visual`

**`--from-angle <path>`:** Skip research stage. Uses the angle's contrast, one-liner, and `image_concept` (for `post`-format angles) to drive image generation and caption. See `2026-05-06-viral-content-engine-design.md` Section 8.3 for details.

**Platform behavior:**
- **Instagram:** Always generates image + caption
- **LinkedIn:** Always generates image + caption
- **X/Twitter:** Caption only (`post.md`), no image generated

## Brand Photo Module

New brand module: `vault/brand/modules/photo.md`

```yaml
---
module: photo
photo_path: vault/assets/me.png
description: "<auto-generated description of appearance>"
last_updated: 2026-05-06
---
```

When the user uploads a photo via `/brand-voice --module photo`, Claude describes their appearance and stores both the image path and a text description. The description is used in GPT-image-2 prompts; the image is sent as a reference for the edit endpoint.

## Image Generation

### Smart Photo Decision

The command analyzes the post topic and decides which approach produces a better-looking image:

**Use reference photo (edit endpoint)** when:
- The concept involves the creator physically — action poses, costumes, speaking on stage, holding objects
- Example: "AI is a superpower" → creator in superhero costume with AI emblem

**Use description only (generate endpoint)** when:
- The concept is abstract, metaphorical, or would look awkward with a real photo
- Example: "5 AI tools you need" → illustrated style with figure matching description

### Generation Details

- **Model:** `gpt-image-2`
- **Master size:** `1024x1024` (square)
- **Text overlays:** Baked into the GPT-image-2 prompt — no manual post-processing
- **Brand elements in prompt:** Colors (`#C5F135` lime, `#111111` black), watermark text (`@oyegpt`), style cues from `style.md`

### Text Mode

When `--mode text`, generates a branded text card instead of a creative image:
- Brand colors as background (`#111111` bg, `#C5F135` accent)
- Key quote or hook text rendered by GPT-image-2
- Same platform adaptation as visual mode

### Platform Adaptation

Deterministic transformations using Pillow — no additional API calls:

| Platform | Output Size | Method |
|----------|------------|--------|
| Instagram | 1080x1080 | Direct resize from master |
| LinkedIn | 1200x627 | Pad top/bottom with `#111111` background + thin `#C5F135` accent bar at top edge |
| X/Twitter | — | No image (text only) |

Padding uses brand colors from `brand.md`.

## Caption Output

`post.md` with platform-specific sections:

```markdown
## LinkedIn
<professional tone, longer form, line breaks for readability>
<hashtags>

## Instagram
<punchy, emoji-friendly, CTA from brand cta.md>
<hashtags>

## X
<280 chars max, hook-driven, 1-2 hashtags max>
```

Brand integration:
- Tone from `style.md` (direct, no-fluff, slapstick energy)
- CTA from `cta.md` (comment-keyword-to-DM for Instagram, adapted per platform)
- Niche context from `niche.md` for relevance

## Session Output Structure

```
vault/outputs/posts/2026-05-06-ai-is-superpower/
├── research.md              # Source material / research
├── image.png                # Master image (1024x1024)
├── image-instagram.png      # 1080x1080
├── image-linkedin.png       # 1200x627
├── post.md                  # Platform-tailored captions
```

X/Twitter gets no image file — just the caption section in `post.md`.

## Publishing Integration

`scripts/publish.py` needs a new content type detection path:
- Detects single-image post by presence of `image.png` (not `1.png` for carousel, not `final.mp4` for reel)
- Picks platform-specific image: `image-instagram.png` for Instagram, `image-linkedin.png` for LinkedIn
- Extracts caption from `post.md` using `## <Platform>` section headers
- X/Twitter: posts text only from `## X` section
